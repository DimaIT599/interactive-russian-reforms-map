# Интерактивная карта реформ и реформаторов России XVIII–XX вв.

Статический учебный frontend-проект на `HTML`, `CSS` и `JavaScript`, который отображает исторические реформы на карте Яндекса, загружает данные из `reforms.json` и позволяет фильтровать их по реформатору, типу и периоду.

## Что умеет проект

- показывает реформы на интерактивной карте Яндекс.Карт;
- загружает данные из `reforms.json`;
- открывает балун и карточку реформы по клику на метку;
- фильтрует реформы по реформатору, типу и периоду;
- обновляет карту и список реформ при изменении фильтров.

## Структура проекта

- `index.html` — основная страница проекта;
- `style.css` — стили интерфейса;
- `script.js` — логика загрузки данных, карты и фильтров;
- `reforms.json` — исторические данные о реформах;
- `.env` — локальный файл с API-ключом Яндекс.Карт, не коммитится;
- `.env.example` — пример файла `.env`;
- `config.js` — локальный файл с конфигурацией для браузера, не коммитится;
- `config.example.js` — пример файла `config.js`;
- `generate-config.js` — небольшой скрипт, который создаёт `config.js` из `.env`;
- `.gitignore` — исключает `.env` и `config.js` из Git;
- `README.md` — инструкция по запуску и использованию.

## Как вставить ключ Яндекс.Карт

Браузер не умеет читать `.env` напрямую, поэтому ключ сначала хранится в `.env`, а затем переносится в `config.js` с помощью генератора.

### Вариант 1. Рекомендуемый: генерация `config.js` из `.env`

1. Создайте файл `.env` на основе примера:

```bash
cp .env.example .env
```

2. Вставьте ваш ключ в строку:

```env
YANDEX_MAPS_API_KEY=ваш_ключ_сюда
```

3. Сгенерируйте `config.js`:

```bash
node generate-config.js
```

После этого рядом с `index.html` появится файл `config.js`, который подключается в браузере.

### Вариант 2. Ручное создание `config.js`

Если нужно, можно создать `config.js` вручную на основе примера:

```bash
cp config.example.js config.js
```

Затем замените значение ключа в `config.js`:

```js
window.APP_CONFIG = {
  YANDEX_MAPS_API_KEY: "ваш_ключ_сюда"
};
```

## Важно по Git

- `.env` не должен попадать в репозиторий;
- `config.js` тоже не должен попадать в репозиторий;
- в `.gitignore` уже добавлены:

```gitignore
.env
config.js
```

## Как запустить проект локально

Запускайте сайт не через `file://`, а через локальный HTTP-сервер, иначе `fetch` для `reforms.json` может работать некорректно.

1. Убедитесь, что `config.js` уже создан.
2. Запустите сервер:

```bash
python -m http.server 8000
```

3. Откройте сайт в браузере:

```text
http://localhost:8000
```

## Как протестировать

Проверьте по шагам:

1. Страница открывается без ошибок загрузки `reforms.json`.
2. Карта Яндекса отображается после вставки API-ключа.
3. На карте видны метки реформ.
4. По клику на метку открывается балун и обновляется карточка реформы справа.
5. Фильтры по реформатору, типу и периоду уменьшают число меток и записей в списке.
6. Кнопка `Сбросить` возвращает полный набор реформ.

## Формат `reforms.json`

Проект ожидает массив объектов. Текущую структуру данных можно сохранять в таком виде:

```json
{
  "id": 1,
  "name": "Отмена крепостного права",
  "reformer": "Александр II",
  "year": 1861,
  "period": "XIX век",
  "type": "социальная",
  "region": "Российская империя",
  "coordinates": [55.7558, 37.6173],
  "goal": "Освободить крестьян от крепостной зависимости.",
  "measures": "Крестьяне получили личную свободу и право выкупать землю.",
  "results": "Началось развитие капиталистических отношений, но сохранились выкупные платежи."
}
```

### Обязательные поля

- `id` — уникальный идентификатор;
- `name` — название реформы;
- `reformer` — имя реформатора;
- `year` — год или период;
- `period` — век или исторический период;
- `type` — тип реформы;
- `region` — регион или территория;
- `coordinates` — массив `[широта, долгота]`;
- `goal` — цель реформы;
- `measures` — основные меры;
- `results` — последствия реформы.

## Полезные замечания

- Если вы изменили `.env`, запустите `node generate-config.js` ещё раз, чтобы обновить `config.js`.
- Если карта не загружается, чаще всего проблема в API-ключе или в том, что `config.js` ещё не создан.
- Исторические данные в `reforms.json` можно расширять, если сохраняется указанная структура полей.
